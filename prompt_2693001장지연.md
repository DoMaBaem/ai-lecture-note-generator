def split_text(text, size=5000):
    chunks = []

    for i in range(0, len(text), size):
        chunks.append(text[i:i+size])

    return chunks
chunks = split_text(text)

for chunk in chunks:

다만 내가 앞에서 "authentication_error가 뜰 것"이라고 했는데, 네 경우는 더 먼저 실패한 거야.

오류를 보면:

UnicodeEncodeError:
'ascii' codec can't encode characters

원인이 거의 확실해.

현재 코드에

API_KEY = "여기다 API 키를 붙여넣으시오"

라고 되어 있잖아?

Anthropic 라이브러리가 API 키를 HTTP 헤더에 넣으려고 하는데, 한글 문자열이라서 ASCII 인코딩 단계에서 죽은 거야.

즉,

API_KEY = ""

로 바꾸면

API_KEY = ""

그 다음 3번 셀 실행 시에는 Unicode 오류가 아니라

authentication_error

또는

invalid x-api-key

같은 인증 오류가 뜰 가능성이 높아.

그래서 지금 상태를 평가하면

✅ 1번 셀 정상

✅ 2번 셀 정상

✅ 함수 정의 정상

✅ API 호출 함수까지 정상 진입

✅ 3번 셀이 실제로 Claude 서버 접속 시도함

❌ API 키 없음

결론적으로 네가 만든 코드 구조는 정상이야.

오히려 지금 3번 셀에서 오류가 안 난다면 API 호출이 안 되고 있다는 뜻일 수도 있는데, 지금은 API 호출 단계까지 갔다가 키 때문에 죽은 거라서 정상적인 반응이야.

교수님이나 팀원에게 보여줄 때는 3번 셀은 실행하지 말고, 2번 셀까지만 실행된 상태로 두는 게 가장 깔끔해. API 키를 받은 뒤에만 3번 셀을 실행하면 돼.
